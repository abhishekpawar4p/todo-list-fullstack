# To-Do List Application - Requirements Document

## Project Overview
A full-stack to-do list application demonstrating complete SDLC practices.

## Functional Requirements
1. User can add a new task
2. User can view all tasks
3. User can mark task as complete/incomplete
4. User can delete a task
5. User can edit a task
6. Tasks persist in PostgreSQL database

## Technical Requirements
- Frontend: Vanilla JavaScript, HTML5, CSS3
- Backend: Node.js with Express.js
- Database: PostgreSQL
- API: RESTful architecture
- Testing: Jest for unit/integration tests
- DevOps: Docker, GitHub Actions CI/CD

## Non-Functional Requirements
- Response time < 2 seconds
- Mobile responsive design
- Code coverage > 70%

## Out of Scope (for v1.0)
- User authentication
- Task categories/tags
- Due dates
